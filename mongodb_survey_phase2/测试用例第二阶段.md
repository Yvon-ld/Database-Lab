## 3. 第二阶段核心测试

## 测试 1：独立题目创建测试

### 目标
验证题目可以脱离问卷单独创建并保存到题库系统中。

### 操作步骤
1. 注册用户 `teacher_a`
2. 登录后进入 `Question Bank`
3. 点击 `New Question`
4. 创建题目：

```text
标题：你的年龄
说明：用于验证题目独立创建
类型：single_choice
选项：
- 18以下
- 18-25
- 26-35
- 36以上
```

### 预期结果
- 创建成功
- 页面跳转到题目详情页
- 可看到：
  - Root ID
  - 当前版本 `v1`
  - Version History 中有 `v1`

---

## 测试 2：题目新版本创建测试

### 目标
验证同一题目可以创建多个版本，且版本链关系正确。

### 操作步骤
1. 在题目详情页点击 `Create New Version`
2. 创建新版本：

```text
标题：你的年龄（2026版）
说明：新增年龄区间
类型：single_choice
选项：
- 18以下
- 18-25
- 26-35
- 36以上
- 40以上
```

### 预期结果
- 创建成功
- 新题目版本为 `v2`
- Root ID 与 v1 相同
- Parent version 指向上一版本
- Version History 中至少有：
  - v1
  - v2

---

## 测试 3：题库创建与加题测试

### 目标
验证可以创建题库并把题目加入题库。

### 操作步骤
1. 在 `Question Bank` 页面创建题库：

```text
题库名：基础人口统计题库
说明：常用基础信息题目
```

2. 在 `你的年龄（2026版）` 详情页中加入该题库

### 预期结果
- 题库创建成功
- 题库列表中出现该题库
- 题目详情页可显示已加入题库
- 问卷构建页右侧题库面板中可看到该题目

---

## 测试 4：题目共享测试

### 目标
验证一个用户创建的题目可以共享给另一个用户。

### 操作步骤
1. 使用 `teacher_a` 创建题目并进入详情页
2. 共享给用户 `student_b`
3. 退出当前账号
4. 注册并登录 `student_b`
5. 进入 `Question Bank`

### 预期结果
- `student_b` 可以看到被共享题目
- 共享题目可以用于新问卷导入
- 共享不会改变原题目的 Root ID

---

## 测试 5：问卷导入题库题测试

### 目标
验证问卷可以导入题库中现有题目版本。

### 操作步骤
1. 登录 `teacher_a`
2. 进入 `New Survey`
3. 填写：

```text
问卷标题：第二阶段导入验证问卷
说明：用于验证题库导入
```

4. 在右侧 `Question Bank` / `All Accessible Versions` 中点击：

```text
Add v2 - 你的年龄（2026版）
```

### 预期结果
- 问卷中出现该题
- 页面显示：

```text
Imported from question bank root ..., version 2
```

- 题目定义字段只读：
  - 标题不可编辑
  - 描述不可编辑
  - 类型不可编辑
  - 选项不可编辑
- 仍允许配置 Survey Jump Rules

---

## 测试 6：浏览器端保存问卷测试

### 目标
验证浏览器交互下可正常保存问卷。

### 操作步骤
1. 进入 `New Survey`
2. 创建一个纯自定义问卷：

```text
标题：浏览器保存验证问卷
题目：你是否满意当前系统？
类型：single_choice
选项：
- 满意
- 不满意
```

3. 点击 `Save Survey`

### 预期结果
- 保存成功
- 跳转到 `/surveys/:id/edit`
- 数据库中存在该问卷
- 仪表盘可以看到该问卷

### 本轮验证结果
- 已通过

---

## 测试 7：已发布问卷锁定测试

### 目标
验证问卷发布后不可再编辑，防止历史问卷内容被污染。

### 操作步骤
1. 创建问卷：

```text
标题：发布锁定验证问卷
题目：你是否愿意继续使用？
选项：愿意 / 不愿意
```

2. 保存问卷
3. 在仪表盘点击 `Publish`
4. 再次打开 `/surveys/:id/edit`

### 预期结果
- 页面显示锁定提示
- `Save Survey` 按钮消失
- 标题输入框禁用
- 已发布问卷不可修改

### 本轮验证结果
- 已通过

## 测试 10：题目版本隔离测试

### 目标
验证不同问卷可同时使用同一题目的不同版本。

### 操作步骤
1. 问卷 A 导入 `你的年龄` v1
2. 问卷 B 导入 `你的年龄（2026版）` v2
3. 发布两个问卷
4. 查看各自编辑页和填写页

### 预期结果
- 问卷 A 仍显示 v1 内容
- 问卷 B 显示 v2 内容
- 两个问卷互不影响

## 4. 第一阶段功能回归测试

## 测试 11：综合问卷功能测试

### 问卷 A：综合功能验证问卷

用于回归：

- 创建问卷
- 单选跳转
- 数字跳转
- 文本/数字校验
- 提交与统计

### 配置

```text
标题：综合功能验证问卷
说明：用于测试单选、多选、文本、数字、跳转和统计功能
允许匿名填写：否
允许重复提交：否
截止时间：7天后
```

### 题目设计

#### Q1 单选题
```text
题目：你是否参加过校内社团？
选项：参加过 / 没参加过
必答：是
跳转：
- 参加过 -> Q2
- 没参加过 -> Q4
```

#### Q2 多选题
```text
题目：你参加过哪些类型的社团？
选项：学术类 / 体育类 / 文艺类 / 公益类
限制：至少1个，最多2个
必答：是
```

#### Q3 文本题
```text
题目：请简要说明你参加社团的主要收获
限制：最少5字，最多50字
必答：是
```

#### Q4 数字题
```text
题目：你平均每周用于课外活动的时间（小时）
限制：0-40，必须整数
必答：是
跳转：
- >=10 -> Q6
- <10 -> Q5
```

#### Q5 文本题
```text
题目：如果时间较少，你认为主要原因是什么？
限制：最少4字，最多40字
必答：是
```

#### Q6 单选题
```text
题目：你是否愿意继续参加校园活动？
选项：愿意 / 不愿意 / 视情况而定
必答：是
```

### 预期结果
- 跳转逻辑正确
- 提交成功
- 统计页正常
