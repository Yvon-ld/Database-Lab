# 关键逻辑说明

## 1. 注册与登录
- 使用 `users` 集合保存用户
- 密码不明文保存，而是保存 `passwordHash`
- 登录后把 `userId` 放到 session 中

## 2. 问卷构建逻辑
创建/编辑页面把问卷配置组装成 JSON：
- 问卷标题
- 问卷说明
- 设置
- questions 数组

后端收到后进行：
1. JSON 解析
2. 数据清洗
3. 自动生成 questionId / optionId（如果前端未给）
4. 合法性检查
5. 保存到 MongoDB

## 3. 跳转逻辑
跳转规则不写死在程序里，而是保存在：
- `questions[].jumpRules`

核心思路：
1. 当前题回答完成后
2. 按优先级遍历规则
3. 第一个匹配规则生效
4. 跳到目标题
5. 如果没有规则匹配，则按 questionOrder 顺序跳转

## 4. 校验逻辑
后端统一校验，前端也做一次基础校验。

### 单选
- 必答检查
- 选项合法性检查

### 多选
- 必答检查
- 最少/最多/恰好选择数检查

### 文本
- 最小长度 / 最大长度检查

### 数字
- 数字合法性检查
- 整数限制检查
- 最大最小值检查

## 5. 提交问卷逻辑
提交时：
1. 根据答案计算真正访问过的题目路径
2. 只校验访问过的题
3. 保存回答
4. 保存访问路径 `visitedQuestionIds`

## 6. 统计逻辑
统计时从 `responses` 里读取数据：
- 单选：统计每个选项被选次数
- 多选：统计每个选项被包含次数
- 文本：展示所有文本
- 数字：展示所有值并计算平均值
