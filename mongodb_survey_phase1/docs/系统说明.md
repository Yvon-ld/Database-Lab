# 系统说明

## 1. 技术选型
- 后端：Node.js + Express
- 数据库：MongoDB + Mongoose
- 前端：EJS + 原生 JavaScript + 简单 CSS
- 身份认证：Session

## 2. 为什么这样选
这套方案非常适合本课程项目：
1. MongoDB 适合问卷这种“结构变化大、题型多、逻辑可配置”的业务。
2. Express + EJS 学习成本较低，便于你后续亲自讲解。
3. Session 登录比 JWT 更适合简单课程项目，调试方便。
4. 题目和跳转规则可以直接作为 Survey 文档的一部分保存，天然贴合 MongoDB 文档模型。

## 3. 丰富功能
- 不是把所有内容塞进一个集合，而是拆成 `users`、`surveys`、`responses`
- Survey 中保存 `version`，为第二阶段迭代做准备
- Response 中保存 `surveyTitleSnapshot` 与 `surveyVersion`
- 支持 owner-only 权限控制
- 支持整卷统计与单题统计
- 支持匿名填写与登录填写共存
- 支持是否允许重复提交
- 跳转逻辑不是写死在代码里，而是保存到数据库里
- 前端不是单纯 JSON 文本框，而是有一个可操作的简单问卷构建器

## 4. 第二阶段为什么不用推倒重来
当前结构已经预留了扩展点：
- `Survey.version`
- `settings`
- `questions[].validation`
- `questions[].jumpRules`
- `Response.visitedQuestionIds`
- `Response.answers[]`

第二阶段就算增加：
- 新题型
- 逻辑条件类型
- 更多统计方式
- 分页问卷
- 草稿保存
都可以在原结构上增量修改。
